# v0.1.0 — Student-First Experience shipped

The first end-to-end slice of ZhyJen is live. A brand-new student can
sign up, answer one placement question, and land on a curated Start-here
view that walks them into their first activity.

## What changed for students

- **One question, not a test.** The new placement question ("Have you
  written any code before?") puts a student on the right starting
  Track without scoring them. Three answers map to three starting
  Tracks: `No, I'm new` → **Web Foundations**, `A little` →
  **Make It Interactive**, `Yes, I'm comfortable` → **Real Projects**.
- **A Start-here view, not a feature tour.** New students see their
  Track name, the first three activities in `order`, and a pinned
  "Start" CTA on the first. The moment they open any Activity, the
  regular dashboard takes over.
- **Track names everywhere.** "Phase" is implementation language.
  Students see **Web Foundations → Make It Interactive → Real Projects
  → Share Your Work**. The four-step progression is now a curriculum,
  not a schema field.
- **A progress strip while they're learning the ropes.** Slim pill
  at the top of the regular dashboard that reads "You're on
  **{Track}** — X of Y activities completed" — and disappears the
  moment they advance to the next Track.
- **An advance-unlock toast** the first time they unlock a new Track.
  One-time. Persisted. Never replays.

## What changed for mentors

- **Invitation links.** Mentors get a list of shareable
  `/invitations/:token` URLs on their dashboard. Each link is unique
  and can be used once to join the workspace as a student.
- **A per-student view at `/mentor`.** One row per student with their
  current Track, last activity, status, and last-updated date. The
  current Track is computed by the same rule the dashboard uses
  (`currentTrackForProfile`), so the two views can never disagree.
- **A new Convex `accept` mutation** that's idempotent — a second
  click on the same invite link is a no-op, not a duplicate member.

## What changed under the hood

- **`src/tracks.ts`** is now the single source of truth for Track
  names, the four-track constant, and the `currentTrackForProfile`
  rule. It's tested in isolation (20 unit tests, all green) and used
  by every Track-aware surface: the dashboard, the activities list,
  the mentor view, the invitation page, and the unlock-toast effect.
- **`profiles.experience`** — the ONE allowed schema change to the
  Profile table. Optional union `none | a_little | comfortable`,
  set once during student onboarding, never updated.
- **`invitations.token`** — the OTHER allowed schema change. Random
  UUID per invitation, indexed by `by_token`. The new
  `api.invitations.accept` mutation looks up by `token`, creates the
  workspace member, marks the invitation `accepted`, and returns the
  workspace id.

## What's next

The next milestone candidates, in no particular order:

- A real-time preview on the activity workspace (Monaco → iframe with
  hot-reload) so students see the result of every keystroke.
- A "challenges" mode where completed activities unlock stretch
  goals with their own starter files.
- Mentor-side feedback — short text comments a mentor can leave on
  a student's `activityProgress.notes`.
- A live site at `zhyjen.app` with the new flow wired up. (The app
  builds and runs locally; we just need a Convex deployment to
  exercise it end-to-end against a real backend.)

If any of those matter more to you than the others, drop a vote or a
comment in the **Roadmap** discussion.

## Try it

```bash
cd apps/zhyjen
vp install
vp exec convex dev   # one-time: provisions Convex + pushes schema
vp dev --port 3003   # in another terminal
```

Then open `http://localhost:3003/signup`, answer the placement
question, and walk through Start-here → first activity → regular
dashboard.
